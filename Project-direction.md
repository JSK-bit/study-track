# 学习打卡应用 — 设计文档

> 基于 2026-07-02 讨论整理

---

## 一、项目定位

面向学生的个人学习记录工具，核心价值是帮助记录每日学习、建立节奏感、看到自己的进步。

### 核心功能

| 功能 | 说明 |
|------|------|
| 每日打卡 | 记录每天的学习完成情况 |
| 学习计划 | 制定每日学习计划，勾选完成 |
| 课表视图 | 查看每日课程安排 |
| 学习曲线 | 可视化看到自己的学习变化趋势 |

---

## 二、应用形态

- **PWA（Progressive Web App）**：不加应用商店添加到桌面
- **移动优先**：学生主要在手机上使用，兼顾平板
- **离线可用**：Service Worker 缓存 + IndexedDB 本地存储
- **零构建**：原生 HTML/CSS/JS，打开即用

### 技术选型

| 层 | 选择 | 原因 |
|----|------|------|
| 框架 | 原生 JS | 零构建、轻量 |
| 存储 | IndexedDB | 已实现 db.js，可直接扩展 |
| 图表 | Chart.js（CDN） | 轻量图表库 |
| 图标 | Lucide 风格线性图标 | 简洁、不幼稚 |
| PWA | manifest.json + SW | 现有资源可沿用 |

---

## 三、设计基调

**温暖实用主义**——打开它感到被陪伴，不是被管理。

### 配色体系

**底色（拒绝纯白刺眼）**

- 页面背景：`#f5f0eb`（暖灰米色）
- 卡片背景：`#faf8f5`（暖白）
- 主文字：`#2d2a3d`（深灰紫）
- 次级文字：`#8b8594`（淡紫灰）
- 分割线/边框：`#e8e2da`

**科目颜色 — 按重要度分级**

| 层级 | 适用科目 | 色板 |
|------|---------|------|
| T1 重要 | 高数、专业课、核心必修 | `#e74c3c` `#e8a838` `#7c3aed` `#d946ef` |
| T2 中等 | 选修、实验课 | `#4a6fa5` `#5b9279` `#9b72cf` `#b45309` |
| T3 轻松 | 体育、通识、公共课 | `#a8c4d8` `#e8b4b8` `#c4b7a6` `#b8c5b0` |

课表卡片：左侧 4px 色条标识层级，内容区用中性色文字。

---

## 四、页面结构

**首页 — 今日看板（默认页）**

一屏半展示：今日课表 + 今日计划 + 打卡入口。

**打卡页**

点击科目直接开始计时，或快速勾选已完成，有圆环勾选微动画反馈。

**统计页**

连续打卡天数、学习时长折线图、科目占比、完成率趋势。

**课表管理页**

管理每周课程，设置时间、地点、重要度层级。

---

## 五、关于游戏化

不做显性游戏化。曲线本身就是最强的反馈信号。

---

## 六、交互风格要点

- 卡片圆角 10-12px
- 线性图标，不用 emoji 做主图标
- 卡片间距 12-16px，信息密度中等偏高
